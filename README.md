# sabyasachisabs.github.io

Personal portfolio site (GitHub Pages). The homepage is generated by `generate_portfolio.py` from your profile data and the GitHub API.

## When to refresh the live site

There is **no hourly schedule** anymore. The site updates when you choose one of the options below.

### After you add, remove, or rename GitHub repositories

The project list comes from the API. Refresh when you want the site to match GitHub:

**Option A — GitHub (no local Python needed)**  
1. Open the repo on GitHub: **Actions**  
2. Select **Update portfolio**  
3. Click **Run workflow** → **Run workflow**  
4. Wait for the run to finish. If `index.html` changed, the bot commits **Auto-update portfolio** and Pages redeploys.

**Option B — On your computer**  
```bash
cd /path/to/sabyasachisabs.github.io
python3 generate_portfolio.py   # or: .venv/bin/python generate_portfolio.py
git add index.html
git status                        # confirm only expected changes
git commit -m "Regenerate portfolio"
git push origin main
```

### After you edit `generate_portfolio.py` or `style.css`

Pushing those files to `main` **automatically** runs **Update portfolio** and may commit an updated `index.html` if the generator output changed.

Typical flow:

```bash
# Edit files, then:
python3 generate_portfolio.py    # optional but recommended before commit
git add generate_portfolio.py style.css index.html
git commit -m "Describe your change"
git push origin main
```

### After you only change images under `assets/`

If the HTML already points at the new file, a push is enough. If you changed paths or copy in `generate_portfolio.py`, regenerate and push as above.

### Preview locally

```bash
python3 -m http.server 8080
```

Open http://127.0.0.1:8080/ — refresh the browser after edits (and after running `generate_portfolio.py` if you changed the generator).

### Advanced: trigger via API

`repository_dispatch` with event type `refresh-portfolio` can still trigger this workflow if you automate it later.
